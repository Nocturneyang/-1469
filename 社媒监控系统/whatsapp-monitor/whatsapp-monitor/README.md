# 📡 WhatsApp Monitor — 部署说明

## 🆕 v2.3.1 更新内容（2026-03-24）

### AI 分析全面升级
- **内置业务知识库**：AI 系统提示词内嵌 Itnio SMS 路由运营业务背景、核心术语（CR/SID/VNL/Route）及运营商合作伙伴列表，无需配置即可精准理解消息语境
- **三层判断树**：将模糊规则替换为带信号词的明确判断树，满足任一条件即触发对应级别（🔴高/🟡中/🟢低）
- **few-shot 示例校准**：内置 10+ 真实场景示例，AI 对照示例判断，大幅降低分类错误率
- **多语言翻译强化**：system 消息级别强制要求输出简体中文，覆盖英文、西班牙文、马来文等多语言消息
- **自定义规则重新定位**：从"覆盖"改为"补充"，避免与内置判断树冲突导致分类不稳定
- **兼容更多 AI 模型**：移除 `response_format: json_object`（OpenAI 专属参数），修复 Kimi/Gemini 等模型返回"无法分析"的问题；新增 JSON 容错提取

### Bug 修复
- **修复前端 JS 全部失效**：误删 `</script>` 闭合标签导致整个脚本块无法解析，所有页面函数均未定义
- **修复连接状态不同步**：SSE 建立连接时主动 HTTP 拉取状态，解决 WhatsApp 缓存 session 快速重连后仍显示"正在生成二维码"的问题
- **修复右侧全局设置被遮挡**：群组列表加载大量数据时，右侧面板添加 `sticky` 定位，滚动时始终可见

## 🆕 v2.3 更新内容

### 监控设置页全面重构
- **两列新布局**：左侧内嵌群组列表，右侧全局设置，无需弹窗，所见即所得
- **群组内嵌搜索 + 全选**：支持实时搜索过滤、一键全选（含二次确认）、即时刷新
- **三级监控模式**（优先级：个人 > 群组 > 全局）：
  - 📌 **全局模式**：沿用右侧全局设置，无需额外配置
  - ⚡ **单条模式**：每条消息独立处理，可设冷却时间
  - 🪟 **逐段汇总模式**：群聊沉默后触发 AI 汇总，可设沉默触发时间和最长等待上限
- **个人成员监控**：点击群组行「👤」按钮展开，可为单个成员单独设置监控模式（优先级最高）
- **时间单位选择**：秒 / 分钟 / 小时 / 天，全局、群组、个人均独立设置
- **友好术语**：专业术语全面替换为通俗说法
  - 滑动窗口时长 → **沉默触发**（群聊停止发言后多久开始分析）
  - 最大等待上限 → **最长等待**（群聊持续时的强制触发间隔）
  - 防抖间隔 → **冷却时间**（处理一条消息后的静默期）
- **三级配置优先级**：个人设置 > 群组设置 > 全局设置，互相独立不干扰
- **WhatsApp 未连接提示**：断线时设置页友好提示，不允许误操作

### 技术优化
- 新增 `/api/groups/members` 接口，支持拉取群组成员列表
- SSE 连接建立时主动 HTTP 轮询当前状态，解决缓存 session 快速登录后前端显示"正在生成二维码"的 Bug
- 后端配置结构新增 `personConfigs`（个人级别配置）和 `groupConfigs`（群组级别配置）
- `getEffectiveGroupConfig` 支持三级优先级合并
- 放宽 `windowMaxSec` 上限，支持天级别配置（最大 86400 秒）

## 🔑 第一步：准备密钥（极简配置）

**首次运行启动脚本时，系统会自动生成 `.env` 配置文件（基于模板）。** 您也可以直接将 `.env.example` 复制并重命名为 `.env`。

用文本编辑器打开 `.env` 文件，按需填入 API 密钥信息：

```env
# AI 模型配置 (兼容所有 OpenAI 格式的接口，如 DeepSeek, Kimi, 阿里通义等)
AI_API_URL="https://api.moonshot.cn/v1/chat/completions"
AI_API_KEY="sk-你的Kimi或DeepSeek密钥..."
AI_MODEL_NAME="moonshot-v1-8k"

# 钉钉自定义机器人
DINGTALK_WEBHOOK="https://oapi.dingtalk.com/robot/send?access_token=你的Token"
DINGTALK_SECRET=""   # 机器人设置了「加签」才填，否则留空
```

### 如何获取核心密钥？

**AI 大模型 API Key（用于消息分析）**
1. 访问对应平台（如 [Kimi 开放平台](https://platform.moonshot.cn/) 或 [DeepSeek 开放平台](https://platform.deepseek.com/)）
2. 注册/登录 → 创建应用或 API 密钥
3. 将生成的字符串填入 `AI_API_KEY`

**钉钉 Webhook**
1. 打开钉钉群 → 群设置 → 智能群助手
2. 添加机器人 → 自定义
3. 安全设置选「加签」（推荐）或「关键词」
4. 复制 Webhook 地址，若选了加签则同时复制密钥

## 📱 第三步：扫码登录 WhatsApp

1. 浏览器访问 **http://localhost:3000**
2. 进入「号码状态」页面，等待二维码出现（首次约需 30-60 秒）
3. 手机打开 WhatsApp → 右上角菜单 → **已关联设备** → **关联设备** → 扫描二维码
4. 扫码成功，页面显示「✅ 已连接」，监控开始运行

> 登录信息保存在本地，重启后自动恢复，无需重复扫码。

## ⚙️ 第四步：配置监控规则

打开控制台 http://localhost:3000，进入「**监控设置**」页：

### 🗂️ 群组列表（左侧）

点击「🔄 刷新」拉取全部 WhatsApp 群组，支持搜索过滤。勾选需要监控的群组，为每个群单独设置监控模式：

| 按钮 | 说明 |
|------|------|
| 📌 全局 | 使用右侧全局默认设置 |
| ⚡ 单条 | 每条消息独立推送，可设「冷却时间」 |
| 🪟 逐段 | 群聊沉默后触发 AI 汇总，可设「沉默触发」和「最长等待」 |

点击「👤」按钮可展开个人成员监控配置（**优先级最高**，覆盖群组和全局设置）。

### 🌐 全局默认设置（右侧）

| 设置项 | 说明 |
|--------|------|
| **监控模式** | 全局默认的单条 / 逐段汇总 模式 |
| **沉默触发** | 群聊停止发言后等多久触发 AI 汇总（逐段模式） |
| **最长等待** | 群聊一直活跃时，最多等多久后强制触发一次分析（逐段模式） |
| **冷却时间** | 处理一条消息后，此时段内再来的消息跳过不分析（单条模式） |
| **关键人白名单** | 填写人名或号码前缀，关键人消息无论紧急度均会推送 |
| **AI 分级规则** | 自定义紧急度判断标准（高/中/低） |
| **仅推送重要消息** | 开启后过滤低紧急度消息 |
| **监控自己的消息** | 开启后自己发的消息也会被 AI 分析 |

配置完成后点击「💾 保存配置」，**立即生效，无需重启**。

## ❓ 常见问题

**Q: 首次安装依赖报错 / 下载 Chromium 失败？**
> 建议开启代理后重新执行 `start.bat` / `start.sh`

**Q: 收不到钉钉消息？**
> 1. 检查「环境配置」页中 Webhook 是否正确填写
> 2. 「监控设置」中确认已勾选对应群组
> 3. 确认「仅推送重要消息」是否过滤掉了该消息

**Q: 页面一直显示「正在生成二维码」？**
> 1. 等待 30-60 秒，Puppeteer 启动 Chrome 需要时间
> 2. 若超过 3 分钟未出现，检查任务管理器是否有大量僵尸 Chrome 进程，有则全部结束后重启
> 3. 也可尝试删除 `.wwebjs_cache` 文件夹后重启

**Q: 重启后需要重新扫码吗？**
> 不需要。登录信息保存在 `.wwebjs_auth/` 目录，自动恢复。

**Q: 个人监控和群组监控可以同时设置吗？**
> 可以。优先级：**个人设置 > 群组设置 > 全局设置**。为某成员单独设置后，该成员消息按个人配置处理，群内其他成员按群组或全局配置处理。
