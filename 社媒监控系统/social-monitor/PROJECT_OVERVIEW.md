# 📊 Social Monitor (多平台社媒监控系统) 项目说明

这是一个基于 Node.js 构建的独立后台监控系统，专门用于实时捕获、分析和持久化存储跨平台（目前以 WhatsApp 和 Telegram 为主）群聊的原始数据。系统兼具极高的稳定性和现代化的可视化管理能力。

## 🎯 核心目标与定位
系统采用“全量采集、防撞锁、媒体冷热分离”的架构策略，通过提供稳健的“纯净”数据流底座，为后续的 AI 舆情分析、商机挖掘以及 CRM 客户管理积攒底层核心数据。

## 🛠️ 技术栈与架构特色

**1. 后端生态与采集器：**
*   **运行环境**：Node.js 
*   **核心依赖**：`whatsapp-web.js` + `puppeteer` (WhatsApp 网页端协议抓取)、`node-telegram-bot-api` (TG 官方机器人 API)
*   **Web 服务**：`express` (提供前台数据的接口支撑)
*   **进程调度**：使用 `PM2` 进行多核并发接管，对易崩溃的 WhatsApp 爬虫和轻量 Telegram 爬虫进行进程级别的隔离，实现防死锁与自动重启拉起。

**2. 核心数据架构 (存储层)：**
*   **强事务本源库**：采用 `better-sqlite3` 驱动的 SQLite (WAL 模式)。利用 `UNIQUE(platform, message_id)` 形成极其坚固的防重锁，有效防止网络断线或连发造成的并发重复写入死循环。
*   **多媒体冷热分离**：探针截获的实体图片、视频等媒体文件会被物理深储至 `media/` 本地目录（冷库），而在数据库的 `media_path` 中仅保留映射路径（热库），保证数据库体积的极致轻便与响应速度。
*   **归一化数据清洗**：跨平台抓取的数据会经过脱水提纯，统一转译为包含 `platform`、`message_id`、`group_id`、`sender_name`、`content` 等统一字段及精准时间戳的规范数据字典。

**3. 前端大屏可视化：**
*   附带极具**新拟态 (Neumorphism)** 审美质感的 Web 监控大屏 (`localhost:3000`)。
*   提供【全盘态势 Dashboard】(数据和渠道占比统计) 与【带图聊天数据瀑布流 Feed】(实时聊天记录相册展示)。

## 🚀 启动与运维方式 (Mac 原生友好)
项目为 Mac 用户提供了完全傻瓜式的 `*.command` 执行脚本：
*   **一键启动**：双击 `start.command` 即可挂载所有的底层数据流服务以及 Web 前端支持，自动弹出二维码窗口等待操作。
*   **一键关闭**：双击 `stop.command` 即可优雅退出所有关联进程。
*   **高阶运维**：极客用户也可基于 `ecosystem.config.js` 全面使用 PM2 (`npx pm2 start/status/logs/stop`) 进行底层指令操控。
